# Splttr. // Pro Instagram Creative Toolkit

**Splttr** is a lightweight, high-performance web application designed for creators who need perfect Instagram carousels and aesthetic white-bordered frames without compromising their privacy.

## 🛡️ Privacy First (The "No-Server" Promise)
Unlike most online image tools, Splttr processes **everything** on the client side.
* **No Server Uploads:** Your photos never leave your computer.
* **Local Processing:** Splitting and bordering are handled by your browser's RAM via the HTML5 Canvas API.
* **Zero Tracking:** No cookies, no analytics, no storage.
* **Session Purge:** Closing the tab or hitting "Reset" instantly wipes all image data from your device's memory.

---

## ✨ Features

### 1. Smart Carousel Splitter
* **High-Res Export:** Saves tiles at 1080px (Instagram's gold standard) to prevent compression artifacts.
* **Automatic Recommendations:** Analyzes your image dimensions and suggests the optimal number of tiles to minimize cropping.
* **Safe Zone Overlay:** Toggle a UI simulation to ensure your text/faces aren't covered by Instagram's buttons or handles.
* **Grid Preview:** Switch to "Profile View" to see how your cover tile looks on a 3-column grid.

### 2. Aesthetic Border Tool
* **Proportional Borders:** Adjust thickness (1-25%) based on the image's longest side for a consistent "Gallery" look.
* **Color Palette:** Choose from classic White, Ivory, or Midnight, or use the custom color picker for brand-specific hex codes.
* **Seamless Transfer:** Move images between the Splitter and Border tools with a single click.

## 🛠️ Technical Stack
* **HTML5/JavaScript:** Core logic and Image/Canvas processing.
* **Tailwind CSS:** Modern, responsive UI with native **Dark Mode** support.
* **Lucide Icons:** Clean, vector-based iconography.
* **LocalStorage:** Remembers your Dark/Light mode preference across sessions.

## 🚀 Deployment (GitHub Pages)

This project is designed to be hosted for free on GitHub Pages.
1. Fork or download this repository.
2. Ensure your main file is named `index.html`.
3. Go to **Settings > Pages** in your GitHub repo.
4. Select the `main` branch as your source and click **Save**.
